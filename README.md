# Тестовое задание Прософт-Системы (JavaScript Frontend Middle) v0.1.0

Данное тестовое задание предназначено для проверки навыков разработки клиентской части проекта
(фронтенда) с использованием библиотеки React, умения работы с RESTful API, умения работы с
спецификацией OpenAPI и описанием функциональных требований к системе в виде вариантов
использования.

Для выполнения тестового задания требуется разработать клиентскую часть проекта воображаемого
интернет-магазина книг "Books Shop" с использованием библиотеки React.

## Структура проекта

В таблице приведено описание основных директорий и файлов проекта.

Файл/директория     |Описание
--------------------|------------------------------------------------------
backend/            |Исходный код серверной части проекта.
frontend/           |Исходный код клиентской части проекта.
nginx/nginx.conf    |Конфигурация обратного прокси-сервера.
docs/openapi.yaml   |Спецификация API серверной части проекта.
docs/use-cases.md   |Описание вариантов использования системы.
docker-compose.yaml |Конфигурация Docker Compose для окружения разработки.
.env                |Конфигурация компонентов для окружения разработки.

## Окружение разработки

Окружение разработки может быть использовано для запуска компонентов проекта на вашем рабочем
компьютере. В него включена серверная часть проекта (бэкенд), обратный прокси-сервер и заготовка
для клиентской части проекта (фронтенд). Серверная и клиентская части проекта запускаются в режиме
горячей перезагрузки (hot reloading). Внесение изменений в их исходный код приводит к их
автоматическому перезапуску.

Для развертывания окружения разработки используйте команду `docker compose up` в директории проекта:

```bash
$ sudo docker compose up
```

После развертывания окружения разработки клиентская часть проекта будет доступна по адресу
http://localhost:8080/.

В окружении разработки предопределены два аккаунта: аккаунт обычного пользователя и
аккаунт администратора сайта.

Логин пользователя |Пароль |Роль
-------------------|-------|------
user               |user   |user
admin              |admin  |admin

## Критерии выполнения

Задание считается выполненным, если в клиентской части проекта реализованы следующие функциональные
возможности:

- Аутентификация пользователя с помощью логина и пароля.
- Просмотр общего каталога книг.
- Просмотр страницы выбранной книги.
- Публикация новой книги в каталоге (Администратор).
- Удаление выбранной книги (Администратор).
- Изменение информации о выбранной книге (Администратор).
